Part 1：AI IDE革命 — Cursor & Windsurf

1.1 什么是AI IDE？

传统IDE vs AI IDE:

维度 VS Code (传统) Cursor (AI原生)
代码补全 基于语法 基于语义理解
搜索范围 当前文件/项目 整个代码库 + 文档
重构能力 查找替换 理解意图，跨文件重构
学习曲线 需要记快捷键 自然语言描述即可

1.3 .cursorrules — 项目级AI配置

关键概念: .cursorrules 是项目根目录的配置文件，定义AI如何理解你的项目。

示例 .cursorrules:

# Project Context
This is a Django REST API for e-commerce platform.

## Tech Stack
- Python 3.11, Django 5.0, DRF 3.15
- PostgreSQL, Redis, Celery
- Docker, Kubernetes

## Code Style
- Use type hints everywhere
- Docstrings in Google style
- Test coverage > 80%

## Architecture Rules
- All business logic in services/, not views/
- Use repository pattern for DB access
- Never expose internal IDs in API responses

## Common Patterns
- Authentication: JWT with refresh tokens
- Pagination: Cursor-based for lists
- Error handling: Custom exception classes

效果: AI生成的代码会自动遵循这些规则。

Part 2：Claude Code — Agent开发的终极武器

2.1 什么是Claude Code？

Claude Code是Anthropic推出的命令行Agent工具，可以： - 理解整个代码库 - 自主执行多步骤任务 - 创建/编辑/删除文件 - 运行测试和修复bug - 与Git集成

定位: 比Cursor更强大，适合复杂的开发任务。

Brief description of what this project does.

- Language: TypeScript 5.0

- Testing: Vitest + Playwright

- `pnpm e2e` - Run E2E tests

- Client Components only for interactivity

- Use early returns

Part 3：Specs 是新的源代码

3.1 什么是Specs？

💡 核心洞察 (来自Ravi Mehta): “In the AI age, the spec IS the code.”

传统开发流程:

需求文档 → 设计文档 → 代码 → 测试 → 文档更新
   ↓           ↓        ↓       ↓         ↓
  过时       过时     真相    部分真相   过时

AI时代开发流程:

Specs (规格说明) ─────────────────────────────→ 代码
       ↑                                        │
       └────────────── AI Agent ────────────────┘
                    (保持同步)

3.2 Specs的组成

# Feature Spec: User Authentication

## Objective
Allow users to sign up, log in, and manage sessions securely.

## User Stories
- As a user, I can sign up with email/password
- As a user, I can log in and stay logged in for 30 days
- As a user, I can reset my password via email

## Technical Requirements
- JWT tokens with refresh mechanism
- Password hashing: bcrypt, cost factor 12
- Rate limiting: 5 attempts per minute

## API Contracts
POST /auth/signup
  Request: { email, password }
  Response: { user, token }

POST /auth/login
  Request: { email, password }
  Response: { user, token, refreshToken }

## Edge Cases
- Invalid email format → 400 Bad Request
- Duplicate email → 409 Conflict
- Wrong password → 401 Unauthorized (generic message)

## Security Considerations
- Never return "user not found" (prevents enumeration)
- Log all auth events for audit

3.3 MBA启示

传统模式 AI模式

沟通成本高 沟通成本低

Part 5：动手实验

Lab 1: 设置Cursor项目

  1. 安装Cursor: https://cursor.sh
  2. 创建项目: mkdir my-project && cd my-project
  3. 创建 .cursorrules:
# My Project Rules
- Use TypeScript
- Prefer functional style
- Always handle errors
  1. 测试: 打开文件，按 ⌘+K，输入 “create a function to validate email”

Lab 2: Claude Code初体验

  1. 安装: npm install -g @anthropic/claude-code
  2. 配置API Key: export ANTHROPIC_API_KEY=sk-...
  3. 创建 CLAUDE.md:
# My CLI Tool
A simple CLI for task management.
## Stack: Node.js, Commander.js
  1. 启动: claude
  2. 输入: “Create a CLI with add, list, done commands for tasks”
