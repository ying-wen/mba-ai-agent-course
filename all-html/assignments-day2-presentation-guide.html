第二周课堂汇报指南

📅 汇报时间：第二周课堂
🎯 目标：深入理解AI Agent生态，培养技术洞察力与商业分析能力

汇报形式

要素 要求
时长 每组 5-8 分钟展示
形式 PPT 或 Demo 演示
互动 2-3 分钟 Q&A 环节
人员 每组派 1-2 名代表汇报

汇报建议

  • ✅ 聚焦核心观点，避免面面俱到
  • ✅ 用数据和案例支撑论点
  • ✅ Demo优于纯PPT，实践优于理论
  • ✅ 准备应对 “So What?” 的追问

汇报内容框架

每组从以下三个方向中选择一个进行深度调研：

A. 行业应用调研报告

适合人群：对垂直行业有兴趣或有行业背景的同学

报告结构

  1. 行业背景与痛点 (1页)
    • 行业规模与现状
    • 核心痛点识别（效率/成本/体验）
    • 为什么AI Agent是解法？
  2. Agent应用现状 (2-3页)
    • 至少3个真实案例分析
    • 每个案例包括：公司/产品、应用场景、实现效果、技术方案
    • 案例来源需可追溯
  3. 技术方案分析 (1-2页)
    • 典型技术架构图
    • 关键技术组件
    • 与传统方案的对比
  4. ROI估算 (1页)
    • 成本结构（开发、运维、API调用）
    • 收益量化（效率提升、成本节省、收入增长）
    • 投资回报周期估算
  5. 落地挑战与建议 (1页)
    • 技术挑战
    • 组织挑战
    • 合规与伦理问题
    • 你的建议
  6. 未来展望 (0.5页)
    • 技术演进方向
    • 市场机会预判

B. 竞品对比分析

适合人群：对产品分析和技术选型有兴趣的同学

对比框架

维度 分析要点
产品定位 目标用户、核心场景、价值主张
技术架构 底层模型、工具调用、记忆机制
核心能力 差异化功能、能力边界
定价模式 免费/订阅/API定价、成本结构
生态建设 插件/扩展市场、开发者社区、合作伙伴
优劣势总结 SWOT分析或能力雷达图

推荐对比组合

组合1：个人AI助手

OpenClaw vs Claude Desktop vs Rabbit R1
  • 关注点：本地 vs 云端、通用 vs 垂直、软件 vs 硬件

组合2：开发框架

LangChain vs LlamaIndex vs Haystack
  • 关注点：抽象层级、RAG能力、企业特性、学习曲线

组合3：多Agent框架

AutoGen vs CrewAI vs LangGraph
  • 关注点：协作模式、状态管理、可控性、扩展性

组合4：AI编程工具

Codex CLI vs Claude Code vs Cursor
  • 关注点：交互模式、上下文理解、工作流集成

输出建议

  • 对比表格（核心维度一览）
  • 能力雷达图
  • 选型建议矩阵（场景→推荐产品）
  • 明确给出”如果你是XXX，选择XXX”的建议

C. 技术深度探索

适合人群：有技术背景、希望深入理解原理的同学

推荐课题

课题 核心问题 参考资源

报告结构建议

  1. 背景与动机 - 为什么这个技术重要？
  2. 核心概念 - 关键术语和原理解释
  3. 技术细节 - 架构图、流程图、代码片段
  4. 实验/对比 - 动手测试的结果
  5. 最佳实践 - 你的经验总结
  6. 局限与展望 - 当前技术的不足

评分标准

维度 权重 优秀标准
内容深度 30% 不止于表面，有独立思考和深入分析
数据支撑 25% 关键论点有可靠数据/案例支持，来源可追溯
创新洞察 25% 有独特观点，能提出”别人没想到”的角度
表达清晰 20% 逻辑清晰，重点突出，听众能快速理解

加分项 ⭐

  • 现场Demo演示
  • 原创数据收集/实验
  • 跨领域联系与洞察
  • 有争议性但有理有据的观点

扣分项 ⚠️

  • 大段复制粘贴无消化
  • 核心数据无来源
  • 超时严重（>10分钟）
  • 避重就轻，回避核心问题

FAQ

Q: 可以用AI辅助写报告吗？
A: 可以，但需标注AI辅助部分，核心观点必须是你自己的思考。

Q: Demo演示必须是自己开发的吗？
A: 不必，可以演示现有产品，但需要有自己的分析视角。

Q: 数据找不到怎么办？
A: 明确说明”未找到公开数据”，用合理假设+敏感性分析替代。

“好的研究不在于知道所有答案，而在于问对问题。”
