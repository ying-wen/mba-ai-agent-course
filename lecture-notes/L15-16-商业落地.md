## Part 2：行业应用深度案例

### 案例1：金融 - AI投研

**痛点**：
- 分析师信息过载
- 研报同质化严重
- 时效性要求高

**Agent方案**：
```
数据Agent → 分析Agent → 撰写Agent → 合规Agent
    ↓            ↓            ↓            ↓
 实时数据     深度分析     结构化报告   风险审核
```

**效果**（某券商实际数据）：
- 研报初稿：4小时 → 30分钟
- 覆盖标的：+300%
- 合规问题：-80%

**关键成功因素**：
- 高质量私有数据
- 严格的合规审核
- 渐进式部署（先内部试点）

### 案例2：法律 - 合同审查

**痛点**：
- 合同条款繁多
- 人工审查耗时
- 风险条款易遗漏

**Agent方案**：
```
合同解析Agent → 条款比对Agent → 风险标注Agent
      ↓               ↓               ↓
  结构化提取      vs标准模板      风险等级+建议
```

**代表公司**：Harvey AI（估值$1.5B）

**效果**：
- 审查时间：2小时 → 15分钟
- 风险识别率：+40%
- 律师效率：3x

### 案例3：客服 - Klarna

**背景**：瑞典金融科技公司，每月数百万客服请求

**效果**（2024年数据）：
- 处理量：相当于700名全职客服
- 解决时间：11分钟 → 2分钟
- 客户满意度：持平
- 年节省成本：**$4000万**

**争议**：
- 引发裁员担忧
- CEO公开表示将继续用AI替代员工

## Part 3：ROI分析框架

### AI项目ROI计算公式

```
ROI = (收益 - 成本) / 成本 × 100%

收益 = 效率提升 + 成本节省 + 收入增长
成本 = 开发成本 + 运营成本 + 风险成本
```

### 三情景分析模型

| 情景 | 假设 | ROI预估 |
|------|------|---------|
| **乐观** | 全部目标达成 | 300%+ |
| **基准** | 达成70%目标 | 150%+ |
| **保守** | 达成50%目标 | 50%+ |

**建议**：用基准情景做决策，用保守情景做预算

### 成本结构拆解

| 成本类型 | 占比 | 说明 |
|----------|------|------|
| **API费用** | 30-50% | 按token计费 |
| **开发人力** | 20-30% | 初期投入 |
| **运维监控** | 10-20% | 持续投入 |
| **风险准备金** | 10-15% | 应对意外 |

### ROI计算示例

**场景**：部署AI客服Agent

**成本**：
- API费用：$5,000/月
- 开发：$50,000（一次性）
- 运维：$2,000/月
- 第一年总成本：$50,000 + $84,000 = **$134,000**

**收益**：
- 替代3名客服：$150,000/年（薪资+福利）
- 效率提升：额外$50,000/年价值
- 第一年总收益：**$200,000**

**ROI** = ($200,000 - $134,000) / $134,000 = **49%**

> **注意**：这是课堂演示用的简化案例，实际项目需要更详细的分析。

## Part A：Agent 性能评估框架

### A.1 为什么需要评估？

> "你无法改进你无法测量的东西。" — Peter Drucker

Agent 不同于传统软件，其行为具有**随机性**和**涌现性**。评估框架帮助我们：
- 量化 Agent 能力边界
- 对比不同方案的优劣
- 追踪迭代改进效果
- 满足企业合规要求

### A.2 核心评估指标

#### 四维评估模型

```
┌─────────────────────────────────────────────────────┐
│              Agent 四维评估模型                      │
├─────────────────────────────────────────────────────┤
│                                                     │
│         任务成功率 (Effectiveness)                  │
│              │                                      │
│              │                                      │
│    效率 ─────┼───── 成本                           │
│ (Efficiency) │    (Cost)                           │
│              │                                      │
│              │                                      │
│         用户满意度 (Satisfaction)                   │
│                                                     │
└─────────────────────────────────────────────────────┘
```

| 维度 | 核心指标 | 计算方法 | 目标值参考 |
|------|----------|----------|------------|
| **任务成功率** | 完成率、正确率 | 成功任务数 / 总任务数 | >85% |
| **效率** | 响应时间、步骤数 | 平均时间 / 最优路径比 | <10s / <1.5x |
| **成本** | Token消耗、API费用 | 每任务平均成本 | <$0.10/任务 |
| **用户满意度** | NPS、CSAT | 问卷调研 | NPS >50 |

#### 细分指标详解

**1. 任务成功率（最核心）**

```python
# 任务成功率计算
success_rate = completed_tasks / total_tasks

# 但需要区分：
strict_success = perfectly_completed / total  # 严格成功
partial_success = acceptable_completed / total  # 部分成功
graceful_failure = safe_failed / total  # 优雅失败
```

**2. 效率指标**

| 指标 | 定义 | 重要性 |
|------|------|--------|
| **首次响应时间** | 用户输入到首次输出 | 用户体验 |
| **任务完成时间** | 从开始到完全完成 | 生产力 |
| **步骤效率** | 实际步骤 / 最优步骤 | 资源消耗 |
| **重试率** | 需要重试的任务占比 | 稳定性 |

**3. 成本指标**

```
总成本 = 输入Token成本 + 输出Token成本 + 工具调用成本 + 基础设施成本

每任务成本 = 总成本 / 完成任务数

成本效益比 = 创造价值 / 总成本
```

**4. 用户满意度**

- **NPS (Net Promoter Score)**：推荐意愿（-100 到 +100）
- **CSAT (Customer Satisfaction)**：满意度评分（1-5分）
- **CES (Customer Effort Score)**：使用难易度
- **留存率**：持续使用比例

#### 1. WebArena：网页 Agent 黄金标准

**背景**：CMU 团队 2023 年发布，评估 Agent 在真实网站上执行复杂任务的能力。

**特点**：
- 812 个真实任务（购物、论坛、GitLab 等）
- 真实网站环境（非模拟）
- 端到端评估

### A.5 A/B 测试设计

#### 为什么 Agent 需要 A/B 测试？

Agent 的行为具有随机性，单次测试不足以说明问题。A/B 测试帮助我们：
- 统计显著地比较不同版本
- 控制混淆变量
- 量化改进效果

#### A/B 测试框架

```
┌─────────────────────────────────────────────────────┐
│              Agent A/B 测试框架                      │
├─────────────────────────────────────────────────────┤
│                                                     │
│  1. 假设定义                                        │
│     H0: 新版本与旧版本无差异                        │
│     H1: 新版本成功率更高                            │
│                                                     │
│  2. 样本量计算                                      │
│     基于效应量、显著性水平、统计功效                │
│     建议: 每组至少100-500个任务                     │
│                                                     │
│  3. 随机分配                                        │
│     任务随机分配到 A/B 组                           │
│     避免选择偏差                                    │
│                                                     │
│  4. 数据收集                                        │
│     记录所有指标（成功率、时间、成本）              │
│                                                     │
│  5. 统计分析                                        │
│     卡方检验（成功率）                              │
│     t检验/Mann-Whitney（连续变量）                  │
│                                                     │
│  6. 决策                                            │
│     p < 0.05: 拒绝H0，采用新版本                    │
│     p >= 0.05: 保留旧版本或继续测试                 │
│                                                     │
└─────────────────────────────────────────────────────┘
```

#### 实践建议

| 阶段 | 建议 |

| **测试中** | 监控数据质量、警惕作弊行为 |
| **测试后** | 检查统计假设、考虑实际显著性 |

#### API 集成示例

```python
# 航班搜索 MCP Tool 实现
import amadeus
from mcp import Tool

class FlightSearchTool(Tool):
    name = "search_flights"
    
    def __init__(self):
        self.client = amadeus.Client(
            client_id='YOUR_API_KEY',
            client_secret='YOUR_API_SECRET'
        )
    
    def execute(self, origin, destination, departure_date, 
                return_date=None, passengers=1, cabin_class='economy'):
        """搜索航班"""
        
        # 调用 Amadeus API
        response = self.client.shopping.flight_offers_search.get(
            originLocationCode=origin,
            destinationLocationCode=destination,
            departureDate=departure_date,
            returnDate=return_date,
            adults=passengers,
            travelClass=cabin_class.upper(),
            max=10
        )
        
        # 格式化返回结果
        flights = []
        for offer in response.data:
            flights.append({
                "price": offer['price']['total'],
                "currency": offer['price']['currency'],
                "airline": offer['validatingAirlineCodes'][0],
                "segments": self._parse_segments(offer['itineraries']),
                "duration": offer['itineraries'][0]['duration']
            })
        
        return {"flights": sorted(flights, key=lambda x: float(x['price']))}
```

```python
        """
```

```python
def detect_and_resolve_conflicts(facts):
    """检测并处理信息冲突"""
    
    conflicts = []
    for i, fact_a in enumerate(facts):
        for fact_b in facts[i+1:]:
            if is_contradictory(fact_a, fact_b):
                conflicts.append((fact_a, fact_b))
    
    resolved = []
    for fact_a, fact_b in conflicts:
        # 策略1: 选择更可信的来源
        if fact_a.credibility > fact_b.credibility:
            resolved.append(fact_a)
        # 策略2: 选择更新的信息
        elif fact_a.date > fact_b.date:
            resolved.append(fact_a)
        # 策略3: 标注争议
        else:
            resolved.append(mark_as_disputed(fact_a, fact_b))
    
    return resolved
```

```python
class CitationManager:
    """引用管理器"""
    
    def __init__(self):
        self.sources = []
        self.citation_map = {}
    
    def add_source(self, url, title, date, credibility):
        """添加来源"""
        source_id = len(self.sources) + 1
        self.sources.append({
            "id": source_id,
            "url": url,
            "title": title,
            "date": date,
            "credibility": credibility
        })
        self.citation_map[url] = source_id
        return source_id
    
    def cite(self, claim, source_url):
        """生成内联引用"""
        source_id = self.citation_map.get(source_url)
        return f"{claim} [{source_id}]"
    
    def generate_bibliography(self):
        """生成参考文献"""
        bib = "## 参考来源\n\n"
        for source in self.sources:
            bib += f"[{source['id']}] {source['title']}. "
            bib += f"访问日期: {source['date']}. "
            bib += f"URL: {source['url']}\n\n"
        return bib
```

```python
class DeepResearchAgent:
    """简化版 DeepResearch Agent"""
    
    def __init__(self):
        self.llm = ChatOpenAI(model="gpt-4o")
        self.tools = [
            Tool(name="web_search", func=self.web_search),
            Tool(name="fetch_page", func=self.fetch_page),
            Tool(name="save_finding", func=self.save_finding),

        self.findings = []

        plan = self._generate_plan(topic)

                results = self.web_search(query)
                for result in results[:3]:  # 每个query取前3
                    content = self.fetch_page(result['url'])
                    finding = self._extract_finding(content, topic)
                    if finding:
                        self.save_finding(finding)
        
        # 3. 生成报告
        report = self._generate_report(topic, self.findings)
        
        return report
    
    def _generate_plan(self, topic):
```

```python
        "shadow_to_assist": {
            "min_samples": 1000,
            "prediction_accuracy": 0.85,
            "coverage": 0.70
        },
        "assist_to_lead": {

            "min_samples": 10000,

        }
```

```python
class AgentROICalculator:
    """Agent项目ROI计算器"""
    
    def calculate(self, params):
        """计算ROI"""
        
        # 直接收益
        labor_savings = (
            params.replaced_fte * 
            params.annual_salary * 
            (1 + params.benefits_ratio)
        )
        
        efficiency_gains = (
            params.existing_staff *
            params.efficiency_improvement *
            params.annual_salary *
            params.hours_saved_ratio
        )
        
        error_reduction = (
            params.annual_error_cost *
            params.error_reduction_rate
        )
        
        total_benefits = labor_savings + efficiency_gains + error_reduction
        
        # 一次性成本
        one_time_costs = (
            params.development_cost +
            params.integration_cost +
            params.training_cost
        )
        
        # 年度持续成本
        annual_costs = (
            params.api_cost_monthly * 12 +
            params.maintenance_fte * params.annual_salary +
            params.monitoring_cost
        )
        
        # 风险调整
        risk_adjusted_benefits = total_benefits * (1 - params.failure_probability)
        risk_cost = params.failure_probability * params.failure_impact
        
        # 计算
        year1_roi = (risk_adjusted_benefits - annual_costs - one_time_costs) / (annual_costs + one_time_costs)
        year2_roi = (risk_adjusted_benefits - annual_costs) / annual_costs
        
        # 回收期
        payback_months = one_time_costs / (risk_adjusted_benefits / 12 - annual_costs / 12)
        
        return {
            "year1_roi": f"{year1_roi:.1%}",
            "year2_roi": f"{year2_roi:.1%}",
            "payback_months": f"{payback_months:.1f}",
            "npv_3yr": self._calculate_npv(risk_adjusted_benefits, annual_costs, one_time_costs, 3),
```
