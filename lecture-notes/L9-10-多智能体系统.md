## Part 1：为什么需要多Agent？

### 单Agent的瓶颈

当任务变得复杂，单个Agent会遇到三个根本性问题：

**1. 上下文瓶颈**

```
单Agent处理复杂任务:

用户: "帮我做一个完整的市场调研报告"

单Agent尝试:
├── 搜索市场数据 ✓
├── 分析竞争格局 ✓
├── 预测市场趋势 △ (开始疲劳)
├── 撰写执行摘要 △ (质量下降)
├── 设计数据图表 ✗ (上下文溢出)
└── 最终审核校对 ✗ (已经混乱)
```

随着任务推进，上下文不断膨胀，模型的注意力被稀释，输出质量下降。

**2. 角色冲突**

让同一个Agent既"创作"又"审核"，就像让同一个人既写代码又做Code Review——很难保持客观性。

**3. 长链推理脆弱**

多步骤任务中，每一步都有出错概率。如果10步任务每步95%准确率，最终准确率只有60%。

### 多Agent的优势

```
多Agent协作:

用户: "帮我做一个完整的市场调研报告"

Coordinator (协调者)
    │
    ├── Researcher (研究员)
    │   └── 负责数据收集和分析
    │
    ├── Analyst (分析师)
    │   └── 负责竞争格局和趋势
    │
    ├── Writer (撰稿人)
    │   └── 负责报告撰写
    │
    └── Reviewer (审核员)
        └── 负责质量把控
```

**优势**：
- 专业分工，各司其职
- 独立上下文，不会溢出
- 可并行执行，效率更高
- 交叉验证，质量更好

### 何时使用多Agent？

| 场景 | 单Agent | 多Agent |
|------|---------|---------|
| 简单问答 | ✅ | 过度设计 |
| 单一技能任务 | ✅ | 不必要 |
| 需要多视角验证 | ❌ | ✅ |
| 复杂多步骤工作流 | 勉强 | ✅ |
| 长时间运行任务 | 不稳定 | ✅ |

> **MBA视角**：多Agent系统就是"虚拟团队管理"。设计多Agent架构，本质上是在设计组织结构。

## Part 2：四种协作模式

### 模式总览

| 模式 | 结构 | 适用场景 | 典型应用 |
|------|------|----------|----------|
| **Pipeline** | 串行流水线 | 步骤明确、依赖清晰 | 内容生产流程 |
| **Supervisor** | 主管调度 | 需要动态决策 | 复杂研究任务 |
| **Debate** | 辩论对抗 | 需要多视角验证 | 风险评估、决策支持 |
| **Swarm/Blackboard** | 群体协作 | 高度并行、涌现式 | 创意生成、探索性任务 |

### 模式1：Pipeline（流水线）

```
Agent A ──► Agent B ──► Agent C ──► 输出

示例: 研究 → 分析 → 撰写 → 审核
```

**特点**：
- 结构清晰，易于理解和调试
- 每个Agent专注单一任务
- 前一步的输出是后一步的输入

**适用场景**：
- 内容生产（采编审发）
- 数据处理（采集清洗分析）
- 审批流程（起草审核发布）

**KPI设计**：
- 每个节点的处理时间
- 每个节点的质量得分
- 整体端到端耗时

### 模式2：Supervisor（主管调度）

```
         Supervisor
              │
    ┌─────────┼─────────┐
    ↓         ↓         ↓
 Worker A  Worker B  Worker C
```

**特点**：
- Supervisor决定任务分配
- 可以动态调度（根据结果决定下一步）
- 支持重试和错误恢复

**Anthropic的实践**：

在[How We Built Our Multi-Agent Research System](https://www.anthropic.com/engineering/multi-agent-research-system)中，Anthropic分享了他们的多Agent研究系统：

- **Lead Agent**：负责规划和协调
- **Subagents**：执行具体的搜索和分析任务
- 每个Subagent可能消耗数万tokens，但只返回精炼的摘要（1000-2000 tokens）

这种架构实现了"关注点分离"——详细的搜索上下文被隔离在子Agent中，主Agent只需要处理精炼后的结果。

### 模式3：Debate（辩论）

```
Agent A ◄───辩论───► Agent B
    │                   │
    └───────┬───────────┘
            ▼
         裁判Agent
            │
            ▼
          结论
```

**特点**：
- 正反方Agent各持立场
- 通过辩论暴露问题和盲点
- 裁判Agent综合判断

**典型应用**：
- 投资决策（多空分析）
- 风险评估（乐观vs悲观）
- 方案比选（A方案vs B方案）

**防跑偏机制**：
- 设置最大辩论轮数
- 要求引用具体证据
- 裁判需要给出判断理由

### 模式4：Swarm/Blackboard（群体协作）

```
         ┌──────────────┐
         │  Blackboard  │
         │  (共享状态)   │
         └──────┬───────┘
                │
    ┌───────────┼───────────┐
    │           │           │
 Agent A    Agent B    Agent C
    │           │           │
    └───────────┴───────────┘
         (并发读写)
```

**特点**：
- 所有Agent共享一个状态空间
- 每个Agent独立判断何时行动
- 结果涌现自群体协作

**适用场景**：
- 创意生成（多角度brainstorm）
- 探索性研究（不确定路径）
- 复杂系统模拟

**挑战**：
- 状态一致性管理
- 死锁和活锁风险
- 结果不确定性

### 模式选择决策树

```
任务结构是否清晰？
    │
    ├── 是 → 步骤是否有严格依赖？
    │         │
    │         ├── 是 → Pipeline
    │         └── 否 → 需要动态调度？
    │                   │
    │                   ├── 是 → Supervisor
    │                   └── 否 → Pipeline并行化
    │
    └── 否 → 需要多视角验证？
              │
              ├── 是 → Debate
              └── 否 → Swarm/Blackboard
```

## Part 3：Agent间通信

### 两种通信范式

| 方式 | 做法 | 优点 | 缺点 |
|------|------|------|------|
| **消息传递** | Agent之间发送消息 | 解耦、易追踪 | 需要协议设计 |
| **共享状态** | 读写共享数据结构 | 灵活、信息丰富 | 一致性管理复杂 |

### 消息格式设计

好的消息格式应该包含：

```json
{
  "from": "researcher",
  "to": "analyst",
  "type": "research_result",
  "content": {
    "topic": "中国咖啡市场",
    "findings": [...],
    "confidence": 0.85,
    "sources": [...]
  },
  "metadata": {
    "timestamp": "2026-03-01T00:10:00Z",
    "token_cost": 3500
  }
}
```

### 状态管理

对于复杂任务，需要跟踪：

```python
class TaskState:
    goal: str           # 最终目标
    plan: List[Step]    # 执行计划
    progress: dict      # 当前进度
    artifacts: dict     # 产出物
    errors: List[Error] # 错误记录
```

## Part 4：框架选型

### 主流框架对比

| 框架 | 开发者 | 特点 | 适用场景 |
|------|--------|------|----------|
| **AutoGen** | Microsoft | 对话驱动、人机协作强 | 研究、原型 |
| **CrewAI** | 开源社区 | 角色定义清晰、易上手 | 团队协作模拟 |
| **LangGraph** | LangChain | 状态机、可视化、生产级 | 复杂工作流 |

### AutoGen

```python
# 概念示例
from autogen import AssistantAgent, UserProxyAgent

assistant = AssistantAgent("assistant", llm_config=config)
user_proxy = UserProxyAgent("user", human_input_mode="NEVER")

# Agent之间通过对话协作
user_proxy.initiate_chat(assistant, message="分析苹果财报")
```

**特点**：
- Agent之间通过消息对话
- 支持人在回路（Human-in-the-loop）
- 代码执行能力强

**局限**：
- 复杂流程控制较难
- 状态管理不够直观

### CrewAI

```python
# 概念示例
from crewai import Agent, Task, Crew

researcher = Agent(
    role="市场研究员",
    goal="收集市场数据",
    backstory="10年市场研究经验..."
)

analyst = Agent(
    role="数据分析师",
    goal="分析市场趋势",
    backstory="专注于消费行业..."
)

task1 = Task(description="调研咖啡市场", agent=researcher)
task2 = Task(description="分析竞争格局", agent=analyst)

crew = Crew(agents=[researcher, analyst], tasks=[task1, task2])
result = crew.kickoff()
```

**特点**：
- 角色定义清晰（Role/Goal/Backstory）
- 任务依赖管理
- 类人团队协作

### LangGraph

```python
# 概念示例
from langgraph.graph import StateGraph

# 定义状态
class ResearchState(TypedDict):
    topic: str
    research: str
    analysis: str
    report: str

# 构建图
workflow = StateGraph(ResearchState)
workflow.add_node("research", research_agent)
workflow.add_node("analyze", analyze_agent)
workflow.add_node("write", write_agent)

workflow.add_edge("research", "analyze")
workflow.add_conditional_edges("analyze", should_continue)
workflow.add_edge("write", END)
```

**特点**：
- 显式状态管理
- 条件分支和循环
- 支持持久化和恢复
- 可视化调试

### 选型建议

| 需求 | 推荐框架 |
|------|----------|
| 快速原型 | CrewAI（最简单） |
| 研究探索 | AutoGen（灵活） |
| 生产部署 | LangGraph（可靠） |
| 自定义需求 | 自研（完全控制） |

## Part 5：企业案例

### 案例：多Agent投研系统

**业务场景**：券商需要自动化生成行业研报

**架构设计**：

```
┌─────────────────────────────────────────────────────┐
│                    PM Agent                          │
│                  (项目经理)                          │
└──────────────────────┬──────────────────────────────┘
                       │
         ┌─────────────┼─────────────┐
         ↓             ↓             ↓
    ┌─────────┐   ┌─────────┐   ┌─────────┐
    │  Data   │   │ Analyst │   │ Writer  │
    │  Agent  │   │  Agent  │   │  Agent  │
    └────┬────┘   └────┬────┘   └────┬────┘
         │             │             │
         └─────────────┴─────────────┘
                       │
              ┌────────┴────────┐
              │  Review Agent   │
              │   (合规审核)     │
              └─────────────────┘
```

**分工**：
- **PM Agent**：接收任务、制定计划、协调进度
- **Data Agent**：搜索新闻、爬取数据、查询数据库
- **Analyst Agent**：财务分析、竞争分析、趋势预测
- **Writer Agent**：结构化撰写、格式化输出
- **Review Agent**：合规检查、事实核验、质量评分

**效果**：
- 研报初稿时间：4小时 → 30分钟
- 数据覆盖：+300%
- 合规问题：-80%

## Part 6：落地挑战与最佳实践

### 常见挑战

| 挑战 | 表现 | 解决方案 |
|------|------|----------|
| **协调开销** | Agent之间沟通成本高 | 减少不必要的交互 |
| **错误传播** | 一个Agent出错影响全链 | 错误隔离、重试机制 |
| **调试困难** | 不知道问题出在哪 | 完善日志、可视化追踪 |
| **成本失控** | Token消耗过高 | 预算控制、摘要压缩 |

### 最佳实践

**1. 从简单开始**

```
不要一开始就设计5个Agent
先用2个Agent验证核心流程
逐步添加复杂度
```

**2. 明确职责边界**

```
每个Agent应该：
✅ 有明确的输入和输出
✅ 有清晰的成功标准
✅ 能独立测试
```

**3. 设计观测指标**

```
每个Agent跟踪：
- 调用次数
- 平均耗时
- Token消耗
- 成功率
- 输出质量分
```

**4. 人在回路**

```
关键决策点保留人工确认
- 高风险操作
- 低置信度输出
- 异常情况
```

## 本讲总结

### 核心概念

- **多Agent系统 = 虚拟团队**：设计Agent架构就是设计组织结构
- **四种协作模式**：Pipeline/Supervisor/Debate/Swarm
- **通信方式**：消息传递 vs 共享状态
- **框架选型**：AutoGen(灵活)/CrewAI(简单)/LangGraph(生产)

### 关键决策点

| 问题 | 选择依据 |
|------|----------|
| 单Agent还是多Agent？ | 任务复杂度、是否需要多视角 |
| 哪种协作模式？ | 任务结构、依赖关系 |
| 哪个框架？ | 团队能力、生产要求 |

### MBA关键洞察

1. **多Agent是组织问题，不只是技术问题**
2. **协调成本是真实成本**，不要过度设计
3. **从简单开始**，逐步增加复杂度
4. **可观测性决定可维护性**

### A.2 AutoGen（微软）

**设计哲学**：对话即协作（Conversation as Collaboration）

```python
# AutoGen 核心概念示例
from autogen import AssistantAgent, UserProxyAgent, GroupChat, GroupChatManager

# 定义专家Agent
researcher = AssistantAgent(
    name="Researcher",
    system_message="你是一位资深市场研究员，擅长数据收集和分析。",
    llm_config={"model": "gpt-4"}
)

analyst = AssistantAgent(
    name="Analyst", 
    system_message="你是一位金融分析师，擅长财务建模和估值。",
    llm_config={"model": "gpt-4"}
)

# 用户代理（可执行代码）
user_proxy = UserProxyAgent(
    name="User",
    human_input_mode="TERMINATE",  # NEVER / ALWAYS / TERMINATE
    code_execution_config={"work_dir": "workspace"}
)

# 群聊协作
group_chat = GroupChat(
    agents=[user_proxy, researcher, analyst],
    messages=[],
    max_round=10
)
manager = GroupChatManager(groupchat=group_chat)

# 启动对话
user_proxy.initiate_chat(manager, message="分析特斯拉Q4财报")
```

**核心特点**：

| 特性 | 说明 |
|------|------|
| **对话驱动** | Agent之间通过自然语言消息协作 |
| **代码执行** | UserProxyAgent可在本地执行Python代码 |
| **人机协作** | 灵活的human_input_mode控制 |
| **群聊模式** | 多Agent在同一对话中协作 |
| **函数调用** | 支持工具注册和调用 |

**适用场景**：
- 研究和原型验证
- 需要代码执行的任务
- 人机协作场景
- 学术研究和探索

**局限性**：
- 复杂流程控制较难
- 状态管理不够显式
- 调试和追踪较困难

### A.3 AgentScope（阿里巴巴）

**设计哲学**：灵活可扩展的分布式智能体框架

```python
# AgentScope 核心概念示例
import agentscope
from agentscope.agents import DialogAgent, DictDialogAgent
from agentscope.pipelines import SequentialPipeline
from agentscope.message import Msg

# 初始化
agentscope.init(model_configs="./model_configs.json")

# 定义Agent
researcher = DialogAgent(
    name="Researcher",
    sys_prompt="你是一位市场研究员...",
    model_config_name="gpt-4"
)

writer = DialogAgent(
    name="Writer",
    sys_prompt="你是一位专业撰稿人...",
    model_config_name="gpt-4"
)

# Pipeline编排
pipeline = SequentialPipeline([researcher, writer])

# 执行
msg = Msg(name="user", content="调研中国咖啡市场")
result = pipeline(msg)
```

**核心特点**：

| 特性 | 说明 |
|------|------|
| **分布式支持** | 原生支持多机分布式部署 |
| **消息机制** | 统一的Msg消息格式 |
| **Pipeline** | 灵活的流程编排 |
| **服务化** | Agent可作为独立服务部署 |
| **可视化** | AgentScope Studio可视化工具 |

**独特功能**：

```python
# 分布式Agent部署
from agentscope.server import RpcAgentServerLauncher

# 启动远程Agent服务
launcher = RpcAgentServerLauncher(
    host="0.0.0.0",
    port=12345,
    agent_class=DialogAgent,
    agent_kwargs={...}
)
launcher.launch()

# 客户端调用
from agentscope.agents import DistAgent

    name="RemoteResearcher",
```

### A.4 LangGraph（LangChain）

**设计哲学**：图即流程，状态即真相（Graph as Workflow, State as Truth）

```python
# LangGraph 核心概念示例
from typing import TypedDict, Annotated
from langgraph.graph import StateGraph, END
from langgraph.prebuilt import ToolExecutor
import operator

# 定义状态Schema
class ResearchState(TypedDict):
    topic: str
    research_data: str
    analysis: str
    report: str
    iteration: int

# 定义节点函数
def research_node(state: ResearchState) -> dict:
    """研究节点：收集数据"""
    topic = state["topic"]
    # 调用LLM进行研究
    research = llm.invoke(f"Research: {topic}")
    return {"research_data": research, "iteration": state["iteration"] + 1}

def analyze_node(state: ResearchState) -> dict:
    """分析节点：分析数据"""
    data = state["research_data"]
    analysis = llm.invoke(f"Analyze: {data}")
    return {"analysis": analysis}

def write_node(state: ResearchState) -> dict:
    """撰写节点：生成报告"""
    analysis = state["analysis"]
    report = llm.invoke(f"Write report: {analysis}")
    return {"report": report}

# 条件函数：是否需要继续研究
def should_continue(state: ResearchState) -> str:
    if state["iteration"] >= 3:
        return "analyze"
    if "需要更多数据" in state.get("analysis", ""):
        return "research"
    return "analyze"

# 构建图
workflow = StateGraph(ResearchState)

# 添加节点
workflow.add_node("research", research_node)
workflow.add_node("analyze", analyze_node)
workflow.add_node("write", write_node)

# 添加边
workflow.set_entry_point("research")
workflow.add_conditional_edges(
    "research",
    should_continue,
    {"research": "research", "analyze": "analyze"}
)
workflow.add_edge("analyze", "write")
workflow.add_edge("write", END)

# 编译运行
app = workflow.compile()
result = app.invoke({"topic": "AI Agent市场", "iteration": 0})
```

### B.1 辩论模式（Debate Pattern）

**核心思想**：通过对立观点的碰撞，获得更全面的分析。

```
┌──────────────────────────────────────────────────────────┐
│                    Debate Pattern                         │
├──────────────────────────────────────────────────────────┤
│                                                           │
│   ┌─────────────┐                 ┌─────────────┐        │
│   │   Bull      │◄───辩论轮次───►│   Bear      │        │
│   │  (看多方)   │                 │  (看空方)   │        │
│   └──────┬──────┘                 └──────┬──────┘        │
│          │                               │               │
│          └───────────┬───────────────────┘               │
│                      ▼                                   │
│               ┌─────────────┐                            │
│               │   Judge     │                            │
│               │  (裁判员)   │                            │
│               └──────┬──────┘                            │
│                      │                                   │
│                      ▼                                   │
│                 最终结论                                  │
│                                                           │
└──────────────────────────────────────────────────────────┘
```

**代码示例（伪代码）**：

```python
from typing import List
from dataclasses import dataclass

@dataclass
class DebateMessage:
    role: str  # "bull" | "bear" | "judge"
    content: str
    evidence: List[str]
    round: int

class DebateSystem:
    def __init__(self, topic: str, max_rounds: int = 3):
        self.topic = topic
        self.max_rounds = max_rounds
        self.history: List[DebateMessage] = []
        
        # 初始化Agent
        self.bull = Agent(
            system_prompt=f"""你是看多方辩手。
            针对主题"{topic}"，你需要：
            1. 收集支持看多的证据
            2. 反驳对方的看空论点
            3. 每个论点必须有数据支撑"""
        )
        
        self.bear = Agent(
            system_prompt=f"""你是看空方辩手。
            针对主题"{topic}"，你需要：
            1. 收集支持看空的证据
            2. 指出看多方忽视的风险
            3. 每个论点必须有数据支撑"""
        )
        
        self.judge = Agent(
            system_prompt="""你是公正的裁判。
            你需要：
            1. 评估双方论点的质量
            2. 指出逻辑漏洞
            3. 给出平衡的最终结论"""
        )
    
    def run_debate(self) -> str:
        # 第一轮：各自陈述
        bull_opening = self.bull.invoke(f"请陈述看多{self.topic}的理由")
        bear_opening = self.bear.invoke(f"请陈述看空{self.topic}的理由")
        
        self.history.append(DebateMessage("bull", bull_opening, [], 1))
        self.history.append(DebateMessage("bear", bear_opening, [], 1))
        
        # 辩论轮次
        for round in range(2, self.max_rounds + 1):
            # Bull反驳Bear
            bull_rebuttal = self.bull.invoke(
                f"请反驳看空方的观点：{bear_opening}"
            )
            self.history.append(DebateMessage("bull", bull_rebuttal, [], round))
            
            # Bear反驳Bull
            bear_rebuttal = self.bear.invoke(
                f"请反驳看多方的观点：{bull_opening}"
            )
            self.history.append(DebateMessage("bear", bear_rebuttal, [], round))
            
            bull_opening = bull_rebuttal
            bear_opening = bear_rebuttal
        
        # 裁判总结
        debate_summary = self._format_history()
        verdict = self.judge.invoke(
            f"请基于以下辩论内容给出最终判断：\n{debate_summary}"
        )
        
        return verdict
    
    def _format_history(self) -> str:
        return "\n\n".join([
            f"[Round {m.round}] {m.role.upper()}: {m.content}"
            for m in self.history
        ])

# 使用示例
debate = DebateSystem("特斯拉股票2026年走势", max_rounds=3)
conclusion = debate.run_debate()
```

**适用场景**：
- 投资决策（多空分析）
- 风险评估
- 方案比选
- 政策分析

**设计要点**：
- 设置最大辩论轮数（防止无限循环）
- 要求引用具体证据
- 裁判需要保持中立
- 输出结构化结论

```python
from dataclasses import dataclass

class TaskStatus(Enum):
    PENDING = "pending"
    IN_PROGRESS = "in_progress"
    COMPLETED = "completed"
    FAILED = "failed"

@dataclass

    agent_name: str

            if r.status == TaskStatus.COMPLETED

        final_output = self.integrator.invoke(
```

**核心思想**：组织架构式的管理层级，适合大规模复杂任务。

```
│                    Hierarchy Pattern                      │
├──────────────────────────────────────────────────────────┤
│                                                           │
│                    ┌─────────────┐                        │

│                    └──────┬──────┘                        │
│                           │                               │
│         ┌─────────────────┼─────────────────┐            │
│         │                 │                 │            │
│         ▼                 ▼                 ▼            │
│   ┌───────────┐    ┌───────────┐    ┌───────────┐       │
│   │  Manager  │    │  Manager  │    │  Manager  │  L1    │
│   │    研发    │    │    市场    │    │    运营    │       │
│   └─────┬─────┘    └─────┬─────┘    └─────┬─────┘       │
```

**代码示例**：

```python
from typing import List, Dict
from abc import ABC, abstractmethod

class HierarchyAgent(ABC):
    def __init__(self, name: str, level: int):
        self.name = name
        self.level = level
        self.subordinates: List['HierarchyAgent'] = []
    
    def add_subordinate(self, agent: 'HierarchyAgent'):
        self.subordinates.append(agent)
    
    @abstractmethod
    def process(self, task: str) -> str:
        pass

class ManagerAgent(HierarchyAgent):
    """管理层Agent：分解任务，分配给下属"""
    
    def __init__(self, name: str, level: int, domain: str):
        super().__init__(name, level)
        self.domain = domain
        self.llm = Agent(
            system_prompt=f"""你是{domain}部门经理。
            职责：
            1. 接收上级任务
            2. 分解为可执行的子任务
            3. 分配给合适的下属
            4. 汇总下属结果向上汇报"""
        )
    
    def process(self, task: str) -> str:
        # 分解任务
        subtasks = self._decompose_task(task)
        
        # 分配给下属执行
        results = []
        for i, subtask in enumerate(subtasks):
            if i < len(self.subordinates):
                result = self.subordinates[i].process(subtask)
                results.append(result)
        
        # 汇总结果
        summary = self.llm.invoke(
            f"请汇总以下下属报告：\n" + "\n".join(results)
        )
        return summary
    
    def _decompose_task(self, task: str) -> List[str]:
        # 使用LLM分解任务
        decomposition = self.llm.invoke(f"将任务分解为子任务：{task}")
        return decomposition.split("\n")  # 简化处理

class WorkerAgent(HierarchyAgent):
    """执行层Agent：实际执行任务"""
    
    def __init__(self, name: str, specialty: str):
        super().__init__(name, level=2)
        self.specialty = specialty
        self.llm = Agent(
            system_prompt=f"你是{specialty}专员，负责执行具体任务。"
        )
    
    def process(self, task: str) -> str:
        return self.llm.invoke(task)

# 构建层级结构
ceo = ManagerAgent("CEO", level=0, domain="全局")

research_manager = ManagerAgent("研发经理", level=1, domain="研发")
market_manager = ManagerAgent("市场经理", level=1, domain="市场")

researcher_1 = WorkerAgent("研究员小王", specialty="市场调研")
researcher_2 = WorkerAgent("研究员小李", specialty="数据分析")
marketer_1 = WorkerAgent("市场专员小张", specialty="内容营销")

# 建立汇报关系
ceo.add_subordinate(research_manager)
ceo.add_subordinate(market_manager)
research_manager.add_subordinate(researcher_1)
research_manager.add_subordinate(researcher_2)
market_manager.add_subordinate(marketer_1)

# 执行
result = ceo.process("制定2026年Q2市场策略")
```

**适用场景**：
- 大型项目管理
- 企业级系统模拟

#### Retrieval（记忆检索）

**三维度加权检索**：

```python
def retrieve_memories(query: str, current_time: datetime) -> List[Memory]:
    """检索与查询相关的记忆"""
    
    memories = get_all_memories()
    scored_memories = []
    
    for mem in memories:
        # 1. 时效性 (Recency): 最近的记忆更重要
        hours_ago = (current_time - mem.timestamp).total_seconds() / 3600
        recency_score = 0.99 ** hours_ago  # 指数衰减
        
        # 2. 重要性 (Importance): 重要的记忆更重要
        importance_score = mem.importance / 10
        
        # 3. 相关性 (Relevance): 与当前情境相关的记忆更重要
        relevance_score = cosine_similarity(
            embed(query), 
            mem.embedding
        )
        
        # 加权综合评分
        final_score = (
            0.5 * recency_score + 
            0.3 * importance_score + 
            0.2 * relevance_score
        )
        
        scored_memories.append((mem, final_score))
    
    # 返回Top-K
    scored_memories.sort(key=lambda x: x[1], reverse=True)
    return [m[0] for m in scored_memories[:20]]
```

### D.2 消息传递（Message Passing）

**特点**：
- Agent之间直接发送消息
- 点对点或广播
- 消息格式需要预定义

**标准消息格式**：

```python
from dataclasses import dataclass
from typing import Optional, Dict, Any
from datetime import datetime
from enum import Enum

class MessageType(Enum):
    REQUEST = "request"      # 请求
    RESPONSE = "response"    # 响应
    NOTIFY = "notify"        # 通知
    BROADCAST = "broadcast"  # 广播

@dataclass
class AgentMessage:
    """标准Agent消息格式"""
    
    # 元数据
    id: str                          # 唯一消息ID
    timestamp: datetime              # 发送时间
    type: MessageType                # 消息类型
    
    # 路由信息
    sender: str                      # 发送者ID
    receiver: str                    # 接收者ID（broadcast时为"*"）
    reply_to: Optional[str] = None   # 回复的消息ID
    
    # 内容
    content: str                     # 消息正文
    structured_data: Optional[Dict[str, Any]] = None  # 结构化数据
    
    # 控制
    priority: int = 5                # 优先级（1-10）
    ttl: int = 3600                  # 存活时间（秒）
    require_ack: bool = False        # 是否需要确认

# 消息总线实现
class MessageBus:
    def __init__(self):
        self.queues: Dict[str, list] = {}
        self.handlers: Dict[str, callable] = {}
    
    def register(self, agent_id: str, handler: callable):
        """注册Agent的消息处理器"""
        self.handlers[agent_id] = handler
        self.queues[agent_id] = []
    
    def send(self, message: AgentMessage):
        """发送消息"""
        if message.receiver == "*":
            # 广播
            for agent_id in self.handlers:
                if agent_id != message.sender:
                    self._deliver(agent_id, message)
        else:
            self._deliver(message.receiver, message)
    
    def _deliver(self, agent_id: str, message: AgentMessage):
        """投递消息"""
        if agent_id in self.handlers:
            self.handlers[agent_id](message)
```

### D.3 共享状态（Shared State）

**特点**：
- 所有Agent读写同一状态空间
- 需要并发控制
- 适合需要全局视图的场景

**实现示例**：

```python
from threading import Lock
from typing import TypedDict, List
import copy

class SharedState(TypedDict):
    """共享状态Schema"""
    topic: str
    research_data: List[str]
    analysis_results: List[str]
    current_phase: str
    agent_status: Dict[str, str]

class StateManager:
    """线程安全的状态管理器"""
    
    def __init__(self):
        self._state: SharedState = {
            "topic": "",
            "research_data": [],
            "analysis_results": [],
            "current_phase": "init",
            "agent_status": {}
        }
        self._lock = Lock()
        self._version = 0
        self._watchers: List[callable] = []
    
    def read(self) -> SharedState:
        """读取状态（返回深拷贝）"""
        with self._lock:
            return copy.deepcopy(self._state)
    
    def update(self, updates: Dict[str, Any], agent_id: str) -> bool:
        """原子更新状态"""
        with self._lock:
            for key, value in updates.items():
                if key in self._state:
                    if isinstance(self._state[key], list):
                        self._state[key].extend(value if isinstance(value, list) else [value])
                    else:
                        self._state[key] = value
            
            # 更新Agent状态
            self._state["agent_status"][agent_id] = "updated"
            self._version += 1
            
            # 通知观察者
            self._notify_watchers()
            return True
    
    def watch(self, callback: callable):
        """注册状态变化监听器"""
        self._watchers.append(callback)
    
    def _notify_watchers(self):
        """通知所有观察者"""
        state_copy = copy.deepcopy(self._state)
        for watcher in self._watchers:
            watcher(state_copy)

# 使用示例
state_manager = StateManager()

# Researcher Agent
def researcher_update():
    data = ["市场规模100亿", "增长率15%"]
    state_manager.update({"research_data": data}, "researcher")

# Analyst Agent监听状态变化
def on_state_change(new_state):
    if len(new_state["research_data"]) > 0:
        # 触发分析
        analysis = analyze(new_state["research_data"])
        state_manager.update({"analysis_results": [analysis]}, "analyst")

state_manager.watch(on_state_change)
```

### D.4 事件驱动（Event-Driven）

**特点**：
- 基于事件发布/订阅
- 松耦合
- 支持异步处理

**实现示例**：

```python
from typing import Callable, List, Dict
from dataclasses import dataclass
from enum import Enum

class EventType(Enum):
    TASK_ASSIGNED = "task_assigned"
    TASK_COMPLETED = "task_completed"
    DATA_READY = "data_ready"
    ERROR_OCCURRED = "error_occurred"
    PHASE_CHANGED = "phase_changed"

@dataclass
class Event:
    type: EventType
    source: str
    payload: Dict[str, Any]
    timestamp: datetime

class EventBus:
    """事件总线"""
    
    def __init__(self):
        self._subscribers: Dict[EventType, List[Callable]] = {}
        self._event_history: List[Event] = []
    
    def subscribe(self, event_type: EventType, handler: Callable):
        """订阅事件"""
        if event_type not in self._subscribers:
            self._subscribers[event_type] = []
        self._subscribers[event_type].append(handler)
    
    def publish(self, event: Event):
        """发布事件"""
        self._event_history.append(event)
        
        if event.type in self._subscribers:
            for handler in self._subscribers[event.type]:
                try:
                    handler(event)
                except Exception as e:
                    print(f"Handler error: {e}")
    
    def get_history(self, event_type: EventType = None) -> List[Event]:
        """获取事件历史"""
        if event_type:
            return [e for e in self._event_history if e.type == event_type]
        return self._event_history

# 使用示例
event_bus = EventBus()

# Analyst订阅数据就绪事件
def on_data_ready(event: Event):
    data = event.payload["data"]
    analysis = analyze(data)
    # 发布分析完成事件
    event_bus.publish(Event(
        type=EventType.TASK_COMPLETED,
        source="analyst",
        payload={"analysis": analysis},
        timestamp=datetime.now()
    ))

event_bus.subscribe(EventType.DATA_READY, on_data_ready)

# Researcher发布数据就绪事件
event_bus.publish(Event(
    type=EventType.DATA_READY,
    source="researcher",
    payload={"data": ["market_data_1", "market_data_2"]},
    timestamp=datetime.now()
))
```

### D.6 混合模式实践

```python
class HybridCommunication:
    """混合通信架构"""
    
    def __init__(self):
        # 共享状态用于全局上下文
        self.shared_state = StateManager()
        
        # 消息总线用于Agent间直接通信
        self.message_bus = MessageBus()

        self.event_bus = EventBus()
    
    def setup_agent(self, agent):
        """配置Agent的通信能力"""
        
        # 注册消息处理
        self.message_bus.register(agent.id, agent.handle_message)
        
        # 订阅感兴趣的事件
        for event_type in agent.subscribed_events:
            self.event_bus.subscribe(event_type, agent.handle_event)
        
        # 注入状态访问
        agent.state_reader = self.shared_state.read
        agent.state_writer = lambda updates: self.shared_state.update(
            updates, agent.id
        )
```

### E.2 协调开销（Coordination Overhead）

**问题表现**：

```
单Agent: 
  任务 ──► 完成
  延迟: 1x

多Agent (无优化):
  任务 ──► 分配(0.2x) ──► Agent A(1x) ──► 同步(0.3x) ──► Agent B(1x) ──► 汇总(0.2x)
  延迟: 2.7x

协调开销 = 2.7x / 1x = 2.7倍
```

**解决方案**：

| 策略 | 描述 | 适用场景 |
|------|------|----------|
| **最小化交互** | 减少不必要的Agent间通信 | 所有场景 |
| **批量处理** | 累积多个消息后批量发送 | 高频交互 |
| **异步执行** | 并行执行独立任务 | 无依赖任务 |
| **智能路由** | 根据任务特征直接路由到目标Agent | 大规模系统 |

**代码示例：最小化交互**

```python
class EfficientCoordinator:
    """高效协调器"""
    
    def __init__(self, agents: List[Agent]):
        self.agents = {a.id: a for a in agents}
        self.task_routing = self._build_routing_table()
    
    def _build_routing_table(self) -> Dict[str, str]:
        """预构建任务路由表"""
        return {
            "research": "researcher",
            "analysis": "analyst", 
            "writing": "writer",
            "review": "reviewer"
        }
    
    def process(self, task: str) -> str:
        # 分类任务
        task_type = self._classify_task(task)
        
        # 直接路由，跳过协调
        if task_type in self.task_routing:
            agent_id = self.task_routing[task_type]
            return self.agents[agent_id].process(task)
        
        # 复杂任务才需要协调
        return self._coordinate_complex_task(task)
    
    def _classify_task(self, task: str) -> str:
        """快速任务分类（规则或小模型）"""
        keywords = {
            "research": ["调研", "收集", "搜索"],
            "analysis": ["分析", "评估", "比较"],
            "writing": ["撰写", "写", "总结"],
            "review": ["审核", "检查", "验证"]
        }
        for task_type, kws in keywords.items():
            if any(kw in task for kw in kws):
                return task_type
        return "complex"
```

### E.3 错误传播（Error Propagation）

**问题表现**：

```
正常流程:
Agent A ──► Agent B ──► Agent C ──► 结果
   ✓          ✓          ✓

错误传播:
Agent A ──► Agent B ──► Agent C ──► 错误结果
   ✓          ✗          ✗（基于错误输入）

级联效应: 一个错误影响整条链路
```

**解决方案**：

```python
from typing import Optional, Tuple
from dataclasses import dataclass
from enum import Enum

class ResultStatus(Enum):
    SUCCESS = "success"
    PARTIAL = "partial"
    FAILED = "failed"

@dataclass
class AgentResult:
    """带状态的Agent结果"""
    status: ResultStatus
    output: str
    confidence: float  # 0-1
    errors: List[str]
    metadata: Dict[str, Any]

class ResilientPipeline:
    """容错Pipeline"""
    
    def __init__(self, agents: List[Agent]):
        self.agents = agents
        self.checkpoints: Dict[str, AgentResult] = {}
        self.max_retries = 3
    
    def execute(self, task: str) -> AgentResult:
        current_input = task
        
        for i, agent in enumerate(self.agents):
            result = self._execute_with_retry(agent, current_input)
            
            # 保存检查点
            self.checkpoints[agent.id] = result
            
            # 错误处理策略
            if result.status == ResultStatus.FAILED:
                return self._handle_failure(agent, i, result)
            
            if result.confidence < 0.5:
                # 低置信度，触发验证
                result = self._validate_result(agent, result)
            
            current_input = result.output
        
        return AgentResult(
            status=ResultStatus.SUCCESS,
            output=current_input,
            confidence=self._calculate_overall_confidence(),
            errors=[],
            metadata={"checkpoints": len(self.checkpoints)}
        )
    
    def _execute_with_retry(self, agent: Agent, input: str) -> AgentResult:
        """带重试的执行"""
        last_error = None
        
        for attempt in range(self.max_retries):
            try:
                output = agent.process(input)
                confidence = self._estimate_confidence(output)
                
                return AgentResult(
                    status=ResultStatus.SUCCESS,
                    output=output,
                    confidence=confidence,
                    errors=[],
                    metadata={"attempts": attempt + 1}
                )
            except Exception as e:
                last_error = str(e)
                continue
        
        return AgentResult(
            status=ResultStatus.FAILED,
            output="",
            confidence=0,
            errors=[last_error],
            metadata={"attempts": self.max_retries}
        )
    
    def _handle_failure(self, agent: Agent, index: int, result: AgentResult) -> AgentResult:
        """处理失败情况"""
        
        # 策略1: 降级处理
        if hasattr(agent, 'fallback'):
            return agent.fallback(result)
        
        # 策略2: 跳过当前Agent
        if index < len(self.agents) - 1:
            # 使用上一个检查点的输出继续
            prev_checkpoint = list(self.checkpoints.values())[-1]
            return AgentResult(
                status=ResultStatus.PARTIAL,
                output=prev_checkpoint.output,
                confidence=prev_checkpoint.confidence * 0.8,
                errors=result.errors + [f"Skipped {agent.id}"],
                metadata={}
            )
        
        # 策略3: 完全失败
        return result
```

### E.4 成本控制（Cost Control）

**问题表现**：

```
成本构成:
┌─────────────────────────────────────────────┐
│  多Agent系统Token消耗                       │
├─────────────────────────────────────────────┤
│  单次调用成本 × Agent数量 × 交互轮数 × 任务数  │
│                                             │
│  示例:                                      │
│  - 单次调用: 2000 tokens                    │
│  - Agent数量: 5                             │
│  - 交互轮数: 3                              │
│  - 任务数: 100                              │
│  = 2000 × 5 × 3 × 100 = 3,000,000 tokens    │
└─────────────────────────────────────────────┘
```

**解决方案**：

```python
from dataclasses import dataclass
from typing import Optional

@dataclass
class CostBudget:
    """成本预算"""
    max_tokens: int
    max_api_calls: int
    max_time_seconds: int
    current_tokens: int = 0
    current_calls: int = 0
    start_time: Optional[datetime] = None

class CostController:
    """成本控制器"""
    
    def __init__(self, budget: CostBudget):
        self.budget = budget
        self.budget.start_time = datetime.now()
    
    def can_proceed(self) -> Tuple[bool, str]:
        """检查是否可以继续执行"""
        
        # Token限制
        if self.budget.current_tokens >= self.budget.max_tokens:
            return False, "Token budget exceeded"
        
        # API调用限制
        if self.budget.current_calls >= self.budget.max_api_calls:
            return False, "API call limit exceeded"
        
        # 时间限制
        elapsed = (datetime.now() - self.budget.start_time).total_seconds()
        if elapsed >= self.budget.max_time_seconds:
            return False, "Time limit exceeded"
        
        return True, "OK"
    
    def record_usage(self, tokens: int, calls: int = 1):
        """记录使用量"""
        self.budget.current_tokens += tokens
        self.budget.current_calls += calls
    
    def get_remaining(self) -> Dict[str, int]:
        """获取剩余预算"""
        return {
            "tokens": self.budget.max_tokens - self.budget.current_tokens,
            "calls": self.budget.max_api_calls - self.budget.current_calls,
            "time_seconds": self.budget.max_time_seconds - 
                (datetime.now() - self.budget.start_time).total_seconds()
        }

class CostAwareAgent:
    """成本感知Agent"""
    
    def __init__(self, agent: Agent, cost_controller: CostController):
        self.agent = agent
        self.cost_controller = cost_controller
    
    def process(self, task: str) -> str:
        # 检查预算
        can_proceed, reason = self.cost_controller.can_proceed()
        if not can_proceed:
            raise BudgetExceededError(reason)
        
        # 估算成本
        estimated_tokens = self._estimate_tokens(task)
        remaining = self.cost_controller.get_remaining()
        
        # 成本优化策略
        if estimated_tokens > remaining["tokens"] * 0.5:
            # 使用摘要而非全文
            task = self._summarize_input(task)
        
        # 执行
        result = self.agent.process(task)
        
        # 记录实际使用
        actual_tokens = self._count_tokens(task + result)
        self.cost_controller.record_usage(actual_tokens)
        
        return result
    
    def _estimate_tokens(self, text: str) -> int:
        """估算Token数"""
        return len(text) // 4  # 粗略估算
    
    def _summarize_input(self, text: str) -> str:
        """压缩输入以节省成本"""
        # 使用更便宜的模型做摘要
        return summarize(text, max_length=500)
```

**成本优化策略**：

| 策略 | 节省比例 | 适用场景 |
|------|----------|----------|
| **模型分层** | 30-50% | 简单任务用小模型 |
| **结果缓存** | 20-40% | 相似任务复用 |
| **输入压缩** | 15-25% | 长文本场景 |
| **提前终止** | 10-30% | 满足条件即停 |
| **批量处理** | 10-20% | 减少API开销 |

### E.5 调试困难（Debugging Challenges）

**问题表现**：

```
调试单Agent:
  输入 ──► 处理 ──► 输出
  日志清晰，可追踪

调试多Agent:
  输入 ──► A ──► B ──► C ──► 输出
              │     │
              ▼     ▼
            异步  状态变化
              │     │
              ▼     ▼
           难以追踪因果关系
```

**解决方案：全链路追踪**

```python
import uuid
from datetime import datetime
from typing import Dict, Any, List

@dataclass
class TraceSpan:
    """追踪Span"""
    trace_id: str          # 全链路ID
    span_id: str           # 当前Span ID
    parent_span_id: str    # 父Span ID
    agent_id: str          # Agent标识
    operation: str         # 操作类型
    start_time: datetime
    end_time: Optional[datetime] = None
    status: str = "running"
    input_data: str = ""
    output_data: str = ""
    metadata: Dict[str, Any] = None
    error: Optional[str] = None

class DistributedTracer:
    """分布式追踪器"""
    
    def __init__(self):
        self.spans: Dict[str, List[TraceSpan]] = {}
    
    def start_trace(self) -> str:
        """开始新的追踪"""
        trace_id = str(uuid.uuid4())
        self.spans[trace_id] = []
        return trace_id
    
    def start_span(
        self, 
        trace_id: str,
        agent_id: str,
        operation: str,
        parent_span_id: str = None,
        input_data: str = ""
    ) -> TraceSpan:
        """开始新的Span"""
        span = TraceSpan(
            trace_id=trace_id,
            span_id=str(uuid.uuid4()),
            parent_span_id=parent_span_id or "",
            agent_id=agent_id,
            operation=operation,
            start_time=datetime.now(),
            input_data=input_data[:1000],  # 截断
            metadata={}
        )
        self.spans[trace_id].append(span)
        return span
    
    def end_span(
        self, 
        span: TraceSpan,
        output_data: str = "",
        status: str = "success",
        error: str = None
    ):
        """结束Span"""
        span.end_time = datetime.now()
        span.output_data = output_data[:1000]  # 截断
        span.status = status
        span.error = error
    
    def visualize_trace(self, trace_id: str) -> str:
        """可视化追踪（ASCII）"""
        spans = self.spans.get(trace_id, [])
        if not spans:
            return "No spans found"
        
        output = [f"Trace: {trace_id}\n"]
        output.append("=" * 60 + "\n")
        
        # 按时间排序
        sorted_spans = sorted(spans, key=lambda s: s.start_time)
        
        for span in sorted_spans:
            duration = (span.end_time - span.start_time).total_seconds() \
                if span.end_time else 0
            status_icon = "✓" if span.status == "success" else "✗"
            
            output.append(
                f"{status_icon} [{span.agent_id}] {span.operation} "
                f"({duration:.2f}s)\n"
            )
            if span.error:
                output.append(f"  └── Error: {span.error}\n")
        
        return "".join(output)

# 使用示例
tracer = DistributedTracer()

class TracedAgent:
    """带追踪的Agent包装"""
    
    def __init__(self, agent: Agent, tracer: DistributedTracer):
        self.agent = agent
        self.tracer = tracer
    
    def process(self, task: str, trace_id: str, parent_span_id: str = None) -> str:
        span = self.tracer.start_span(
            trace_id=trace_id,
            agent_id=self.agent.id,
            operation="process",
            parent_span_id=parent_span_id,
            input_data=task
        )
        
        try:
            result = self.agent.process(task)
            self.tracer.end_span(span, output_data=result, status="success")
            return result
        except Exception as e:
            self.tracer.end_span(span, status="failed", error=str(e))
            raise
```
