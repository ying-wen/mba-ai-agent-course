## Part 1：为什么MBA学员需要了解低代码平台？

### 三个核心理由

| 理由 | 说明 | MBA价值 |
|------|------|---------|
| **快速验证想法** | 无需等待开发排期，一天内完成原型 | 降低试错成本 |
| **企业首选方案** | 70%以上的企业AI项目从低代码起步 | 了解主流路径 |
| **技术评估基础** | 理解实现方式，才能评估供应商方案 | 增强决策能力 |

### 低代码 vs 纯代码对比

| 维度 | 低代码平台 | 纯代码开发 |
|------|------------|------------|
| 学习曲线 | 1-3天上手 | 数周至数月 |
| 开发速度 | 小时级 | 天/周级 |
| 灵活性 | 中等（依赖平台功能） | 完全灵活 |
| 维护成本 | 低 | 高 |
| 适用阶段 | MVP/验证期 | 规模化/定制化 |
| 人员要求 | 业务人员可操作 | 需要开发团队 |

### 典型应用场景

**✅ 适合低代码的场景**
- 客服Bot快速上线
- 内部知识库问答
- 文档处理自动化
- 营销内容生成
- 数据分析报告

**⚠️ 可能需要代码的场景**
- 复杂业务逻辑定制
- 大规模高并发系统
- 深度系统集成
- 核心算法自主可控

> **MBA视角**：低代码平台让"想法"到"原型"的周期从月缩短到天。这意味着你可以在投入大量资源前，先用低成本验证方向。

## Part 3：Coze平台详解

### 平台概述

**Coze**（扣子）是字节跳动推出的AI Bot开发平台，特点是：
- 🇨🇳 **中文优先**：界面、文档、社区全中文
- 🔌 **插件丰富**：500+官方插件，覆盖搜索、数据、工具等
- 🚀 **一键发布**：支持飞书、微信、抖音、网页等多渠道
- 💰 **免费可用**：核心功能免费，商业功能按需付费

### 核心概念

| 概念 | 说明 | 类比 |
|------|------|------|
| **Bot** | 完整的对话智能体 | 一个员工 |
| **人设** | Bot的角色定义和行为规则 | 岗位职责 |
| **插件** | 扩展Bot能力的工具 | 专业技能 |
| **知识库** | Bot可查询的私有数据 | 内部资料 |
| **工作流** | 复杂任务的执行流程 | 工作SOP |
| **记忆** | Bot的上下文和长期记忆 | 工作日志 |

### 创建第一个Bot：智能客服

**目标**：创建一个能回答产品问题、引导用户的客服Bot

#### Step 1: 创建Bot

1. 访问 [coze.cn](https://www.coze.cn)
2. 登录后点击「创建Bot」
3. 填写基本信息：
   - 名称：`智能客服小助手`
   - 描述：`7x24在线的产品咨询助手`

#### Step 2: 设置人设与提示词

```
# 角色
你是【公司名】的智能客服，名叫小智。

# 目标
- 解答用户关于产品的问题
- 引导用户完成购买/注册
- 收集用户反馈

# 风格
- 亲切友好，使用简洁的口语化表达
- 遇到复杂问题，先安抚再引导
- 适当使用emoji增加亲和力 😊

# 规则
1. 不确定的问题不要瞎编，承认不知道并引导转人工
2. 涉及价格，以官网为准
3. 涉及隐私，不主动询问敏感信息

# 开场白
你好！我是小智，有什么可以帮您的吗？🙂
```

#### Step 3: 添加知识库

1. 点击「知识库」→「创建知识库」
2. 上传产品文档（支持PDF、Word、网页链接）
3. 等待解析完成
4. 在Bot设置中关联知识库

```
知识库最佳实践：
├── 产品手册.pdf → 功能说明
├── 常见问题.docx → FAQ
├── 价格表.xlsx → 定价信息
└── https://公司官网/帮助中心 → 在线文档
```

#### Step 4: 添加插件

推荐插件配置：

| 插件 | 用途 | 配置 |
|------|------|------|
| 联网搜索 | 回答知识库外的问题 | 开启 |
| 天气查询 | 闲聊场景 | 可选 |
| 图片生成 | 辅助说明 | 可选 |

#### Step 5: 设置工作流（可选）

对于复杂场景，可创建工作流：

```
用户咨询 → 意图识别
           ├── 产品问题 → 查询知识库 → 回复
           ├── 投诉建议 → 记录信息 → 安抚+转人工
           ├── 购买意向 → 推荐产品 → 引导下单
           └── 其他 → 闲聊模式
```

#### Step 6: 测试与发布

1. 右侧「预览」面板测试对话
2. 测试各种场景：正常问答、边界情况、异常输入
3. 点击「发布」→ 选择渠道（网页/飞书/微信等）
4. 按指引完成渠道配置

### Coze进阶功能

| **多Bot协作** | 多个Bot分工合作 | 复杂业务流程 |

## Part 4：Dify平台详解

### 平台概述

**Dify** 是一个开源的AI应用开发平台，特点是：
- 🔓 **开源可控**：完整源码，可私有部署
- 🧠 **RAG强大**：内置向量数据库，知识库能力领先
- 🔧 **灵活架构**：支持几乎所有主流大模型
- 📊 **可观测性**：完整的日志、监控、评估体系

### 部署方式选择

| 方式 | 说明 | 适用 | 成本 |
|------|------|------|------|
| **Dify Cloud** | 官方托管 | 快速体验/小规模 | $0-159/月 |
| **Docker本地** | 本地运行 | 开发测试 | 免费（需硬件） |
| **私有云部署** | 自有服务器 | 企业生产 | 服务器成本 |

### 本地部署（Docker）

```bash
# 1. 克隆仓库
git clone https://github.com/langgenius/dify.git
cd dify/docker

# 2. 启动服务
docker compose up -d

# 3. 访问界面
# 打开 http://localhost:80
# 首次访问设置管理员账号
```

**系统要求**：
- CPU: 2核+
- 内存: 4GB+
- 存储: 20GB+
- Docker & Docker Compose

### 创建应用：知识库问答系统

**目标**：搭建企业内部知识库，支持员工自助查询

#### Step 1: 配置模型

1. 进入「设置」→「模型供应商」
2. 添加模型API Key：
   - OpenAI（GPT系列）
   - Anthropic（Claude系列）
   - 或国内模型（通义、文心等）

```
推荐配置：
├── 对话模型: GPT-4o / Claude 3.5 Sonnet
├── Embedding模型: text-embedding-3-small
└── Rerank模型: cohere-rerank-v3 (可选)
```

#### Step 2: 创建知识库

1. 进入「知识库」→「创建知识库」
2. 上传文档：
   - 支持：PDF, DOCX, TXT, MD, HTML
   - 批量上传：可拖拽多个文件
3. 配置分段：
   - 分段长度：500-1000字符（推荐）
   - 重叠：50-100字符
4. 选择Embedding模型
5. 等待处理完成

```
知识库优化技巧：
1. 文档预处理：清理格式、统一术语
2. 合理分段：按章节/主题切分
3. 添加元数据：来源、时间、分类
4. 定期更新：设置同步策略
```

#### Step 3: 创建对话应用

1. 「工作室」→「创建应用」→「聊天助手」
2. 配置Prompt：

```
# 角色
你是公司内部知识库助手，帮助员工查询公司政策、流程、规范等信息。

# 任务
- 基于知识库内容准确回答问题
- 无法确定时，明确告知并建议咨询相关部门
- 提供信息来源引用

# 约束
- 只回答与公司相关的问题
- 不泄露敏感信息
- 保持客观中立
```

3. 关联知识库：
   - 添加上一步创建的知识库
   - 设置召回参数：Top K = 3-5

#### Step 4: 高级配置

**RAG参数调优**：

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| Top K | 召回文档数量 | 3-5 |
| Score阈值 | 相关性过滤 | 0.5-0.7 |
| Rerank | 重排序 | 建议开启 |
| 引用显示 | 展示来源 | 建议开启 |

**对话配置**：

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| Temperature | 创造性 | 0.3-0.5（知识问答低一些） |
| Max Tokens | 最大输出 | 1000-2000 |
| 上下文窗口 | 历史对话 | 10轮 |

#### Step 5: 发布与集成

**发布方式**：
1. **网页嵌入**：生成iframe代码
2. **API调用**：RESTful API接入现有系统
3. **SDK**：Python/JS SDK

```python
# Python SDK示例
from dify_client import DifyClient

client = DifyClient(api_key="your-api-key")
response = client.chat("公司年假政策是什么？")
print(response.answer)
```

## Part 5：n8n工作流详解

### 平台概述

**n8n** 是一个强大的工作流自动化平台，特点是：
- ⚡ **自动化强大**：700+应用集成，几乎连接一切
- 🔗 **AI原生**：内置LangChain，支持复杂AI工作流
- 🏠 **自托管**：完全私有部署，数据不出网
- 📈 **可扩展**：自定义节点，无限可能

### 核心概念

| 概念 | 说明 | 类比 |
|------|------|------|
| **Workflow** | 自动化流程 | 生产流水线 |
| **Node** | 流程中的操作单元 | 工作站 |
| **Trigger** | 触发工作流的条件 | 开关 |
| **Connection** | 节点间的数据流 | 传送带 |
| **Credential** | 连接外部服务的凭证 | 钥匙 |

### 节点类型

```
触发节点（Trigger）
├── Webhook → 接收外部请求
├── Schedule → 定时执行
├── Email Trigger → 收到邮件时
└── Manual → 手动执行

操作节点（Action）
├── HTTP Request → 调用API
├── AI → ChatGPT/Claude/LangChain
├── Database → 数据库操作
├── Email → 发送邮件
└── 700+ 应用集成...

逻辑节点（Logic）
├── IF → 条件分支
├── Switch → 多路分支
├── Merge → 合并数据
└── Loop → 循环处理
```

### 创建工作流：智能邮件处理

**目标**：自动分析收到的邮件，分类处理，紧急邮件触发提醒

#### Step 1: 部署n8n

```bash
# Docker快速启动
docker run -it --rm \
  --name n8n \
  -p 5678:5678 \
  -v ~/.n8n:/home/node/.n8n \
  n8nio/n8n

# 访问 http://localhost:5678
```

#### Step 2: 创建工作流

1. 新建Workflow
2. 添加触发器：「Email Trigger (IMAP)」
   - 配置邮箱连接
   - 设置检查频率

#### Step 3: 添加AI分析节点

添加「OpenAI」节点：

```
Prompt:
分析以下邮件，返回JSON格式：
{
  "category": "紧急/重要/一般/垃圾",
  "summary": "一句话摘要",
  "action": "需要的操作",
  "priority": 1-5
}

邮件内容：
{{ $json.text }}
```

#### Step 4: 添加条件分支

添加「IF」节点：

```
条件：{{ $json.category }} = "紧急"
├── True → 发送Slack通知 + 邮件标记
└── False → 仅归档
```

#### Step 5: 完整工作流

```
[Email Trigger]
      ↓
[OpenAI: 分析邮件]
      ↓
[IF: 是否紧急?]
   ├── Yes → [Slack: 发送通知]
   │              ↓
   │         [Gmail: 添加标签"紧急"]
   │
   └── No → [IF: 是否垃圾?]
                ├── Yes → [Gmail: 移到垃圾箱]
                └── No → [Gmail: 归档]
```

| 跨系统自动化 | n8n | Zapier | 集成丰富，可自托管 |
| SaaS应用连接 | Zapier | Make | 生态最大 |
| 数据敏感场景 | Dify/n8n自托管 | - | 私有部署 |
| 快速原型验证 | Coze | Zapier | 免费+快速 |
| 复杂AI工作流 | n8n | Dify | LangChain集成 |

### 组合使用策略

有时候一个平台不够，组合使用效果更好：

```
常见组合：

1. Coze + n8n
   └── Coze做对话界面，n8n做后台自动化

2. Dify + Zapier
   └── Dify做AI核心，Zapier连接SaaS应用

3. n8n + Dify
   └── n8n做流程编排，Dify做AI推理

4. All-in-one
   └── Dify（知识库+Agent）→ 大多数场景够用
```

## 附录：常见问题

### Q1: 这些平台会不会很快过时？

不同层次的"过时"：
- **界面变化**：会经常更新，但核心概念稳定
- **平台选择**：头部平台相对稳定（Zapier 10年+）
- **底层能力**：Prompt设计、RAG原理、工作流思维是通用技能

> 建议：掌握原理比记住操作更重要

### Q2: 免费版够用吗？

| 平台 | 免费版适合 | 何时升级 |
|------|------------|----------|
| Coze | 个人使用、原型验证 | 需要API/高级功能 |
| Dify | 开源自托管可完全免费 | 需要云托管/企业支持 |
| n8n | 开源自托管可完全免费 | 需要云托管/企业支持 |
| Zapier | 简单流程（100任务/月） | 流程多/复杂 |

3. 知识库更新机制？
4. 数据存储在哪里？
5. 有没有备选方案？切换成本？
