# L11-12 讲稿：MCP 与工具生态（工具集成标准）

## Slide 1: 标题页
这一讲讲 MCP（Model Context Protocol，模型上下文协议/工具接入协议）。如果说 Agent 要“会用工具”，那 MCP 解决的是：工具如何以标准方式接入，让不同模型、不同框架都能用同一套工具。

## Slide 2: 为什么需要标准
没有标准会发生什么？每个团队都要为每个模型写一套工具接口；工具越多维护越痛苦；安全审计也做不起来。标准的价值是：降低集成成本、提高复用、提升生态规模。

## Slide 3: MCP 的核心思想
你可以把 MCP 类比成企业里的“API 合同”：定义工具有哪些能力、输入输出格式是什么、权限与错误怎么处理。这样模型在调用工具时就更可控。

## Slide 4: 商业视角：生态与平台
有标准就会有生态：更多工具提供方愿意接入；企业可以更快组合出自己的 Agent；平台方可以围绕标准做市场与治理。

## Slide 5: 风险与治理
工具越强，风险越大。MCP 落地要配套：权限最小化、敏感操作审批、日志审计、以及沙盒环境。否则 Agent 可能“做错事”甚至“越权做事”。

## Slide 6: 案例：数据分析 Agent 的工具链
一个数据分析 Agent 可能需要：SQL 查询工具、指标字典、报表导出、邮件发送。用标准协议接入后，你可以快速换模型、换框架，但工具层保持稳定，这对企业非常关键。

## Slide 7: 收尾
一句话：MCP 让工具接入像“插 USB”，减少重复工程，让企业更专注在业务流程与治理，而不是在接口泥潭里消耗。
