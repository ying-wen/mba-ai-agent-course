# L5-6 讲稿：智能体（Agent）设计与实现

## Slide 1: 标题页
这一讲我们讨论 Agent（智能体）：你可以把 Agent 当成“虚拟员工”。LLM 提供语言与推理能力，但真正能在企业里跑起来的，是 Agent：它有目标、有计划、会用工具、能在流程里被管理。

## Slide 2: 为什么 LLM 还不够
只让 LLM 输出文本，很多场景停留在“写作助手”。但企业更需要的是：查数据、填表单、发邮件、建工单、生成报告、触发审批。这些都需要工具调用、权限边界、以及可追踪的执行过程。

## Slide 3: Agent 的基本构件
一个典型 Agent 包含：
- 目标与任务分解：把需求拆成步骤。
- 记忆：保存关键事实与上下文。
- 工具：能访问外部系统。
- 反馈与自检：降低错误。
管理上对应：目标管理、知识管理、资源管理、质量管理。

## Slide 4: 规划（Planning）与执行（Acting）
规划像项目经理，执行像一线员工。好的 Agent 会先规划再执行，并在关键节点做检查点。商业上，这会显著降低“跑偏成本”。

## Slide 5: ReAct 等典型范式
你不需要记住算法细节，只要理解思想：模型在“思考—行动—观察”之间循环。它的价值是：每一步都能被记录、可解释，也更容易插入规则与审计。

## Slide 6: 企业落地的三道门
第一，权限与合规：Agent 能访问什么数据、能做什么动作。第二，可靠性：失败如何回退到人工。第三，成本：工具调用与上下文会把 token 成本放大。

## Slide 7: 案例：财务对账 Agent
财务对账是很适合 Agent 的场景：从多个系统拉取数据、做匹配、标记差异、生成解释报告、并把异常推给负责人复核。它把人的重复劳动压缩到“高价值判断”，ROI 很清晰。

## Slide 8: 收尾
把 Agent 当产品，不要当 demo。你要问：目标是什么、流程怎么走、数据从哪来、风险怎么控、指标怎么衡量。这样 Agent 才能从“炫技”走向“交付”。
